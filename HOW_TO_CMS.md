# Инструкция по работе с CMS (Sanity Studio)

Этот документ объясняет, как самостоятельно редактировать контент сайта: услуги и цены, кейсы "до/после", команду врачей, отзывы и фото/видео клиники — без участия программиста.

## Вход в Studio

Адрес панели управления:

**https://chirkova-dentist.sanity.studio/**

Откройте ссылку, войдите через свой аккаунт (Google или email). Если у вас ещё нет доступа — попросите того, кто настраивал сайт, добавить вас как участника проекта в Sanity.

После входа слева вы увидите 5 разделов:

- **Service** — Услуги и цены
- **Before / After Case** — Кейсы "до/после"
- **Team Member** — Команда (врачи)
- **Review** — Отзывы
- **Clinic Photo / Video** — Фото и видео клиники

## Общие правила для всех разделов

1. **Двуязычность.** Почти у каждого текстового поля есть две версии — украинская и английская (сгруппированы в блоки "uk" и "en"). Сайт показывает нужный язык в зависимости от того, какую версию сайта смотрит посетитель. Заполняйте **обе** версии — если оставить английскую пустой, на английской версии сайта будет пустое место.

2. **Порядок отображения (Display order).** У каждой карточки есть поле "Display order" — число. Чем меньше число, тем выше/раньше карточка показывается на сайте. Если хотите поменять порядок двух услуг местами — поменяйте местами их числа.

3. **Поля со звёздочкой / "Required"** — обязательны. Sanity не даст опубликовать документ, если такое поле пустое — оно подсветится красным.

4. **Save vs Publish.** Studio сохраняет черновик автоматически по ходу ввода, но **на сайт изменения попадают только после нажатия кнопки "Publish"** в правом верхнем углу карточки документа. Не забывайте её нажимать.

5. **После Publish сайт не обновляется мгновенно** — статический сайт собирается заново при следующей пересборке (после изменений). Если нужно увидеть изменения на живом сайте сразу — обратитесь к разработчику, чтобы запустить пересборку (`Run workflow` в GitHub Actions либо `git push`).

## 1. Услуги и цены (Service)

Каждая услуга (Вініри, Імпланти, Елайнери и т.д.) — отдельный документ.

Поля:

| Поле | Описание |
|---|---|
| Display order | Порядок показа карточки услуги |
| Назва послуги / Service name | Название услуги (укр / англ) |
| Короткий опис / Short description | 1-2 предложения, показывается на карточке |
| Ціна на картці (від) / Card price | Цена "от", которая видна на карточке услуги без открытия попапа, напр. "від 4 500 грн" |
| **Full price list (priceItems)** | Список конкретных позиций с ценами — то, что открывается в попапе при клике на карточку |

### Как добавить позицию в прайс-лист услуги

1. Откройте нужную услугу (например "Вініри").
2. Внизу найдите поле **"Full price list"**.
3. Нажмите **"+ Add item"**.
4. Заполните 4 поля: Назва (укр), Name (англ), Ціна (укр, напр. "9 500 грн"), Price (англ, напр. "₴9,500").
5. Нажмите Publish.

Чтобы **удалить** позицию — откройте её внутри списка и нажмите значок корзины/три точки → Remove.

Чтобы **поменять порядок** позиций внутри списка — перетащите их за иконку с точками (⠿) слева от строки.

### Как добавить новую услугу целиком

1. В разделе "Service" нажмите **"+ Create"** (обычно в правом верхнем углу списка).
2. Заполните все поля как описано выше.
3. Добавьте хотя бы одну позицию в Full price list — без этого Sanity не даст опубликовать.
4. Publish.

## 2. Кейсы "до/после" (Before / After Case)

Поля:

| Поле | Описание |
|---|---|
| Display order | Порядок показа |
| Лікування / Treatment | Что делали, напр. "Вініри" |
| Тривалість / Duration | Сколько занял курс, напр. "3 візити" |
| Підпис "До" / "Before" caption | Короткое описание исходной ситуации, напр. "Жовтий, нерівний прикус" |
| Підпис "Після" / "After" caption | Короткое описание результата, напр. "Білі, рівні зуби" |
| Before photo | Фото "до" |
| After photo | Фото "після" |

### Как добавить кейс

1. "+ Create" в разделе "Before / After Case".
2. Заполните Treatment, Duration, подписи для обоих фото (укр/англ).
3. Загрузите фото "до" и "после" — нажмите на поле фото, выберите файл с компьютера.
4. **Hotspot (фокус кадра):** после загрузки фото можно нажать на него ещё раз и указать кружком, какая часть фото самая важная (обычно — рот/зубы). Это нужно, чтобы при обрезке фото под разные экраны важная часть не пропадала.
5. Publish.

## 3. Команда (Team Member)

Поля:

| Поле | Описание |
|---|---|
| Display order | Порядок показа |
| Ім'я | Имя — одно и то же для обоих языков |
| Посада / Role | Должность (укр/англ), напр. "Стоматолог-ортопед" |
| Біографія / Bio | Короткая биография (укр/англ) |
| Роки досвіду | Число — сколько лет опыта |
| Підпис до років / Years label | Текст рядом с числом, напр. "років досвіду" / "years experience" |
| Photo | Фото врача |

### Как добавить / удалить / отредактировать врача

- **Добавить:** "+ Create" → заполнить поля → загрузить фото → Publish.
- **Отредактировать:** откройте карточку врача в списке, измените нужное поле, Publish.
- **Удалить:** откройте карточку → меню (три точки в правом верхнем углу) → **Delete**. Подтвердите удаление.

## 4. Отзывы (Review)

Поля:

| Поле | Описание |
|---|---|
| Display order | Порядок показа |
| Ім'я клієнта | Имя клиента, оставившего отзыв |
| Інціали | Максимум 3 символа, напр. "ОК" — показывается как аватар вместо фото |
| Клієнт з якого року (необов'язково) | Год, с которого клиент у вас, просто для доверия |
| Текст відгуку / Review text | Сам текст отзыва (укр/англ) |

Добавление/удаление — так же, как у врачей: "+ Create" / открыть карточку → удалить через меню.

## 5. Фото / видео клиники (Clinic Photo / Video)

Поля:

| Поле | Описание |
|---|---|
| Display order | Порядок показа |
| Тип / Type | Переключатель Photo / Video — выберите, что загружаете |
| Photo | Появляется, если выбран тип Photo |
| Video file | Появляется, если выбран тип Video — загружается файл видео (mp4 и т.п.) |
| Підпис (необов'язково) / Caption | Необязательная подпись под фото/видео |

### Как добавить фото или видео клиники

1. "+ Create".
2. Выберите тип — Photo или Video.
3. Загрузите соответствующий файл.
4. (Опционально) добавьте подпись.
5. Publish.

## Частые вопросы

**Я нажал(а) Publish, но на сайте ничего не изменилось.**
Это нормально для первых минут — сайт статический, и пересобирается не мгновенно. Если изменений нет дольше часа — напишите разработчику.

**Я случайно удалил(а) документ — можно вернуть?**
В Sanity есть история изменений (History — иконка часов сверху карточки), но восстановление удалённого документа лучше делать сразу — обратитесь к разработчику.

**Можно ли загружать фото в высоком разрешении?**
Да, Sanity сам оптимизирует и сжимает изображения для сайта. Загружайте оригиналы хорошего качества — не нужно сжимать вручную.

**Форма обратной связи на сайте — это тоже часть CMS?**
Нет. Заявки с формы "Швидкий запит" не сохраняются в Sanity — они сразу пересылаются в Telegram-бот клиники. CMS отвечает только за контент (услуги, кейсы, команда, отзывы, медиа), не за заявки клиентов.
